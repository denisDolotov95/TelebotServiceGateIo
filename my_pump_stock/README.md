## my_pump_stock

Telegram‑бот для поиска и отслеживания «pump»‑активов в вашем портфеле на Gate.io. Использует локальный REST‑сервис `gate_rest_api`, хранит статистику и заметки в PostgreSQL и отправляет результаты в Telegram.

---

### Основная идея

- Берёт **список ваших спотовых активов** через `gate_rest_api` (`/spot/spot_accounts`).
- Прогоняет их через **набор алгоритмов** (сейчас `Алгоритм №1` и `Алгоритм №2`), которые:
  - анализируют историю свечей (`/spot/candlesticks`),
  - смотрят стакан заявок (`/spot/order_book`),
  - проверяют тикеры (`/spot/tickers`),
  - отфильтровывают слабые и неинтересные монеты,
  - формируют детальные текстовые отчёты и inline‑клавиатуры.
- Результаты и метрики работы сохраняются в **PostgreSQL** (среднее время выполнения, найденные активы, заметки).
- Все уведомления отправляются в **Telegram** в ваш личный чат.

---

### Возможности

- **Поиск pump‑кандидатов (Алгоритм №1)**
  - Анализирует свечи по каждому активу в формате `CURR_USDT`.
  - Сравнивает цены между `0`‑м и последним (`N-1`) баром.
  - Учитывает:
    - лимит баров `BOT_BAR_LIMIT`,
    - интервал `BOT_BAR_INTERVAL` (например, `5m`),
    - минимальный процент изменения `BOT_PERCENT`.
  - При выполнении условий:
    - подтягивает стакан (`/spot/order_book`),
    - тикеры (`/spot/tickers`),
    - строит текстовый отчёт,
    - добавляет inline‑клавиатуру для быстрого действия.
- **Дополнительный алгоритм (Алгоритм №2)**
  - Анализирует стакан и другие метрики (см. `ConfAlgorithm2`).
- **Фильтрация активов**
  - Исключает нулевые/мелкие балансы, `USDT`, а также валюты из списка `EXCLUDED_SPOTS`.
- **Заметки по монетам**
  - Можно сохранять и просматривать персональные заметки по каждой валюте.
- **Статистика и метрики**
  - В таблице `ArithmeticMean` хранится:
    - количество запусков алгоритма,
    - суммарное время выполнения,
    - среднее время одного прогона (используется в `/drop_time_*` командах).
  - В таблице `AllPumpStocks` аккумулируется информация по найденным активам.
- **Управление ботом и конфигурацией из чата**
  - Доступны команды для:
    - изменения лимита баров, интервала, процента, лимита стакана;
    - запуска алгоритмов вручную;
    - просмотра текущей конфигурации и документации;
    - управления клавиатурой и проверкой состояния.

Все «опасные» команды доступны только владельцу (по `TELEGRAM_CHAT_ID` и/или `CONTAINER_ID`).

### Технологии

- **Python 3.12**
- **pyTelegramBotAPI** (асинхронный режим)
- **aiohttp** — HTTP‑клиент для запросов к `gate_rest_api`
- **gate-api** — официальная Python‑библиотека Gate.io
- **PostgreSQL + SQLAlchemy + asyncpg** — хранение метрик и данных
- **APScheduler** — планировщик периодических запусков алгоритмов
- **pdfkit / prettytable / pandas** — генерация таблиц и вспомогательной аналитики
- Логирование в файл `./log/<CONTAINER_ID>_my_pump_stock.log` с ротацией

### Переменные окружения

Основные переменные задаются через `.env` и файлы `env/*`:

- **База данных**
  - `DB_DRIVER` — драйвер (по умолчанию `postgresql`)
  - `DB_USERNAME`, `DB_PASSWORD` — учётные данные
  - `DB_HOST`, `DB_PORT` — хост и порт PostgreSQL
  - `DB_SERVICE_NAME` — имя базы (например, `gate_io`)
- **Gate.io**
  - `GATE_HOST` — URL API Gate.io
  - `GATE_KEY`, `GATE_SECRET` — API‑ключ и секрет
  - `LOCAL_GATE_REST_API_IP`, `LOCAL_GATE_REST_API_PORT` — адрес локального `gate_rest_api`
- **Telegram**
  - `TELEGRAM_HOST` — хост Telegram API
  - `TELEGRAM_BOT_TOKEN` — токен бота
  - `TELEGRAM_CHAT_ID` — ID чата владельца
  - `TELEGRAM_CONN_TIME` — таймаут подключения
- **Алгоритмы**
  - `BOT_BAR_INTERVAL` — интервал свечей (например, `5m`)
  - `BOT_SCHED_INTERVAL` — период автозапуска Алгоритма №1 (минуты)
  - `BOT_BAR_LIMIT` — лимит баров для анализа
  - `BOT_PERCENT` — минимальный процент изменения цены
  - `BOT_ORDER_BOOK_LIMIT` — лимит заявок в стакане
- **Прочее**
  - `CONTAINER_ID` — ID контейнера/инстанса
  - `BOT_NAME` — имя бота (по умолчанию `my_pump_stock`)

Детали см. в `app/config.py`, `app/algorithm.py`, `docker-compose.yml`.

### Установка и запуск (локально)

```bash
cd my_pump_stock
pip install -r requirements.txt

# настроить переменные окружения: DB_*, TELEGRAM_*, GATE_*, LOCAL_GATE_REST_API_*

python -m app
```

При первом запуске:

- создаются директории `./app/log` и `./app/data`;
- инициализируется соединение с PostgreSQL;
- настраивается планировщик APScheduler (автозапуск Алгоритма №1);
- в Telegram закрепляется сообщение с описанием текущих настроек алгоритмов;
- добавляется клавиатура с командой `/help`.

---

### Запуск в Docker

```bash
cd my_pump_stock
docker compose up -d
```

`docker-compose.yml`:

- собирает образ из `Dockerfile`;
- монтирует volume `gate_io_log` в `/usr/src/app/log`;
- использует env‑файлы:
  - `env/variable.env`
  - `env/<BRANCH>/gate.env`
  - `env/<BRANCH>/telegram.env`
  - `env/<BRANCH>/database.env`

### Структура

```text
my_pump_stock/
├── app/
│   ├── __main__.py            # точка входа, scheduler + polling
│   ├── config.py              # конфиг: DB_*, GATE_*, TELEGRAM_*, параметры алгоритмов
│   ├── algorithm.py           # реализация AsyncAlgorithm, Algorithm1, Algorithm2
│   ├── runners.py             # автозапуск алгоритмов, сохранение метрик и результатов
│   ├── telebot_handler/       # обработчики команд, inline‑клавиатура, заметки
│   ├── database/              # движок, модели, запросы к PostgreSQL
│   ├── util/                  # утилиты, кнопки Telegram, фильтры контента
│   └── gate_wrapper/          # обёртка над gate-api, парсеры JSON
├── docker-compose.yml
├── Dockerfile
├── requirements.in / requirements.txt
└── env/                       # переменные окружения для разных окружений
```

### Команды (общее представление)

Подробный список и формат команд можно посмотреть в `Commands.info` (`config.py`) и в `telebot_handler/message.py`, но в целом доступны:

- команды для запуска алгоритмов (`/algorithm_1`, `/algorithm_2` и т.п.);
- изменение параметров алгоритмов (лимиты баров, интервал, процент, лимит стакана);
- управление заметками (`/note <CUR> <текст>`, просмотр/список заметок);
- список активов (`/list_my_spots`) и их фильтрация;
- служебные команды (`/help`, `/check`, `/pause`, `/active`, `/stop`, `/logs`, `/add_buttons`, `/del_buttons`, `/current_conf`, `/readme` и др.).
