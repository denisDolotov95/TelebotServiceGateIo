## pump_stock

Telegram‑бот для продвинутого анализа спотовых активов на Gate.io. Содержит набор алгоритмов (№1–5), которые ищут разные паттерны роста, всплесков объёма и накопления, сохраняют результаты в БД и отправляют отчёты в Telegram.

---

### Основная идея

- Берёт **актуальный портфель спотовых активов** пользователя через `gate_rest_api` (`/spot/spot_accounts`).
- Для каждого актива прогоняет один или несколько **алгоритмов**:
  - анализируют историю свечей (`/spot/candlesticks`),
  - анализируют стакан (`/spot/order_book`),
  - смотрят тикеры (`/spot/tickers`),
  - фильтруют по объёмам, дельте цены, соотношению `bids/asks` и др.
- Результаты обрабатываются и выводятся:
  - сообщениями в Telegram,
  - либо файлами (`txt/html/pdf`) — в зависимости от настроек `Config.output`.
- В PostgreSQL сохраняются **метрики работы** (среднее время, количество срабатываний) и найденные «интересные» активы.

---

### Алгоритмы (концептуально)

Базовые параметры:

- `BOT_ORDER_BOOK_LIMIT` — лимит заявок в стакане (asks/bids), по умолчанию `30`.
- `calculate_delta(first_price, second_price)` — функция, считающая процентное изменение:

```python
delta = (float(first_price) - float(second_price)) / float(second_price) * 100
```

Далее — упрощённое описание логики каждого алгоритма (детали см. в `app/config.py`, `app/algorithm.py` и `app/telebot_handler`).

#### Алгоритм №1 — рост цены («Восход на гору»)

Ищет **обычный рост цены** монеты на заданный `BOT_PERCENT` между `0`‑м и последним (`BOT_BAR_LIMIT - 1`) баром.

Основные параметры:

- `BOT_BAR_INTERVAL_1` — интервал свечи, например `5m`.
- `BOT_BAR_LIMIT_1` — количество баров для анализа, по умолчанию `100`.
- `BOT_PERCENT_1` — минимальный процент роста между `0` и `BOT_BAR_LIMIT - 1`.
- `BOT_SCHED_INTERVAL_1` — периодичность автозапуска алгоритма (в минутах).

Если `delta >= BOT_PERCENT_1`, бот:

- подтягивает стакан и тикеры;
- формирует отчёт (соотношение объёмов, текущая цена, дельта);
- отправляет результат с inline‑клавиатурой в Telegram.

#### Алгоритм №2 — всплеск объёмов без сильного роста цены («Всплеск объёма»)

Ищет **аномальные всплески объёма** при ограниченном росте цены:

- сравнивает объём сделки в квоте между барами `N` и `N + BOT_BAR_RANGE_2`;
- проверяет, что:
  - объём на `N + BOT_BAR_RANGE_2` существенно больше (`BOT_PRICE_RATIO_2` раз),
  - при этом рост цены не превышает `BOT_PERCENT_2`.

Основные параметры:

- `BOT_BAR_INTERVAL_2` — интервал свечи, например `1h`.
- `BOT_BAR_RANGE_2` — диапазон сравнения баров.
- `BOT_BAR_LIMIT_2` — количество баров.
- `BOT_PERCENT_2` — максимально допустимый процент роста.
- `BOT_PRICE_RATIO_2` — коэффициент роста объёма.

Если выполняются оба условия (объём и цена), алгоритм помечает монету как кандидата и формирует отчёт.

#### Алгоритм №3 — удержание/накопление в диапазоне («Двойное дно / Длительное накопление / Яма»)

Ищет **длительное боковое движение / накопление**:

- количество баров ≈ `BOT_BAR_LIMIT_3`;
- рост цены между `N` и `N + BOT_BAR_RANGE_3` по всему диапазону **не превышает** `BOT_PERCENT_3`;
- при этом разница суммарных объёмов в стакане `bids/asks` превышает `BOT_ORDER_BOOK_RATIO_3`.

Пример параметров:

- `BOT_BAR_INTERVAL_3` — `1d`;
- `BOT_BAR_RANGE_3` — `1`;
- `BOT_BAR_LIMIT_3` — `90`;
- `BOT_PERCENT_3` — `10`;
- `BOT_ORDER_BOOK_RATIO_3` — `5`.

Логика:

- «скользящее окно» по барам (`N` → `N + BOT_BAR_RANGE_3`);
- считаем, сколько раз дельта цены удовлетворяет условию;
- анализируем суммарные `bids/asks` в стакане;
- если цена долго «топчется» в диапазоне, а в стакане перевес по `bids`, монета попадает в отчёт.

#### Алгоритм №4 — мощный всплеск объёма («Всплеск объёма 2.0»)

Ищет **очень большие объёмы за короткий набор баров**:

- в заданном интервале (`BOT_BAR_INTERVAL_4`, например `1d`) проверяет, что:
  - объём `trade_volume_in_base` на баре `N + BOT_BAR_RANGE_4` существенно больше (`BOT_TRADE_RATIO_4` раз), чем на `N`,
  - при этом цена не падает (дельта ≥ 0).

Пример параметров:

- `BOT_BAR_INTERVAL_4` — `1d`;
- `BOT_BAR_RANGE_4` — `1`;
- `BOT_BAR_LIMIT_4` — `10`;
- `BOT_TRADE_RATIO_4` — `100`;
- `BOT_SCHED_INTERVAL_4` — `1440` (1 раз в сутки).

#### Алгоритм №5

Дополнительный алгоритм с собственной конфигурацией в `ConfAlgorithm5` (например, комбинация сигналов предыдущих алгоритмов или особые условия по объёмам/цене). Подробности см. в `app/config.py` и `app/algorithm.py`.

### Технологии

- **Python 3.12**
- **pyTelegramBotAPI** (асинхронный `AsyncTeleBot`)
- **aiohttp** — HTTP‑клиент для работы с `gate_rest_api`
- **gate-api** — обёртка над официальным API Gate.io
- **PostgreSQL + SQLAlchemy + asyncpg** — база данных для статистики и найденных активов
- **APScheduler** — расписание запусков алгоритмов (авто‑режим)
- **pdfkit, prettytable, pandas** — форматирование отчётов и файлов
- Логирование в `./log/<CONTAINER_ID>_pump_stock.log` с ротацией

---

### Переменные окружения (основные)

- **База данных**
  - `DB_DRIVER`, `DB_USERNAME`, `DB_PASSWORD`, `DB_HOST`, `DB_PORT`, `DB_SERVICE_NAME`
- **Gate.io**
  - `GATE_HOST`, `GATE_KEY`, `GATE_SECRET`
- **Telegram**
  - `TELEGRAM_HOST`, `TELEGRAM_BOT_TOKEN`, `TELEGRAM_CHAT_ID`, `TELEGRAM_CONN_TIME`
- **Общие настройки**
  - `CONTAINER_ID` — ID контейнера (используется для логов и доступа)
  - `BOT_FILE_NAME` — имя бота (по умолчанию `pump_stock`)
  - `BOT_ORDER_BOOK_LIMIT` — лимит заявок в стакане (общий для всех алгоритмов)
- **Параметры алгоритмов**
  - `BOT_BAR_INTERVAL_<ID>`, `BOT_BAR_LIMIT_<ID>`, `BOT_PERCENT_<ID>`,
  - `BOT_BAR_RANGE_<ID>`, `BOT_PRICE_RATIO_<ID>`, `BOT_TRADE_RATIO_<ID>`,
  - `BOT_SCHED_INTERVAL_<ID>` — период автозапуска.

Точный список и значения по умолчанию смотрите в `app/config.py` и `docker-compose.yml`.

### Установка и запуск (локально)

```bash
cd pump_stock
pip install -r requirements.txt

# настроить env-файлы: env/variable.env, env/<BRANCH>/gate.env,
# env/<BRANCH>/telegram.env, env/<BRANCH>/database.env

python -m app
```

При старте:

- создаются директории `./app/log` и `./app/data`;
- настраивается планировщик APScheduler (по умолчанию автозапуск Алгоритма 1 и 4);
- в Telegram отправляется и закрепляется сообщение с текущими настройками всех алгоритмов;
- добавляется клавиатура с `/help`.

---

### Запуск в Docker

```bash
cd pump_stock
docker compose up -d
```

`docker-compose.yml`:

- собирает образ из `Dockerfile`;
- монтирует volume `gate_io_log` в `/usr/src/app/log`;
- использует env‑файлы:
  - `env/variable.env`
  - `env/<BRANCH>/gate.env`
  - `env/<BRANCH>/telegram.env`
  - `env/<BRANCH>/database.env`

### Замечания

- Бот рассчитан на **личное использование** (доступ по `TELEGRAM_CHAT_ID`/`CONTAINER_ID`).
- Алгоритмы **не являются торговыми рекомендациями**, а дают сигналы для дополнительного анализа.
- Перед запуском в продакшене рекомендуется протестировать на демо‑аккаунте или с минимальными объёмами.
